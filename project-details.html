Crimson Reign

About

Crimson Reign is a hack-and-slash game set in a gritty medieval world, where players engage in fast-paced melee combat while navigating perilous environments. The game features AI-controlled enemies, immersive combat mechanics, and an interactive world that provides a dynamic experience for players.

Project Info

  • Time Frame: 2 months
  • Engine: Unreal Engine 4.25
  • Genre: Hack-and-Slash
  • Game Mode: Single-player
  • Platform: Windows (PC)

Key Features

  • Combat System The game features a responsive melee combat system, allowing the player to execute various attacks with intuitive user input, paired with smooth animations for each action.
  • Enemy AI AI behavior is powered by Behavior Trees and basic state logic, enabling enemies to patrol, chase, and attack the player based on defined conditions. Navigation meshes facilitate enemy pathfinding.
  • User Interface (UI) The game features a responsive melee combat system, allowing the player to execute various attacks with intuitive user input, paired with smooth animations for each action.

Game Mechanics

  • Damage System A straightforward health and damage system allows players to track their progress and encounters.
  • Timers Timers are used for delayed actions, such as bomb explosions or environmental interactions.
  • Interactive Platforms Floating platforms and triggers are driven by interpolation for smooth transitions.

Programming Concepts Applied

  • Object-Oriented Programming Applied fundamental concepts such as inheritance, polymorphism, and encapsulation to create reusable and extendable gameplay classes (Actors, Characters, etc.).
  • AI Logic Employed Behavior Trees to program enemy AI for patrol, chase, and attack behavior.
  • HUD & AI Designed and implemented basic HUD elements, including health and stamina bars, along with interactive menus.

Challenges & Solutions

  • AI Pathfinding Integrating effective enemy pathfinding with navigation meshes and ensuring smooth navigation during combat and exploration.
  • Combat Feedback Balancing combat feedback (e.g., hit reactions, damage indicators) to ensure a satisfying player experience.
  • Debugging & Optimization Used logging and sphere drawing for debugging gameplay features like AI pathfinding and damage systems to ensure correct behavior during testing.

Acknowledgments

  • Special thanks to Stephen Ulibarri for his Unreal Engine 4 C++ The Ultimate Game Developer Course, which provided the foundation for many of the concepts used in this project.

Apex Ruin

About

Apex Ruin is a third-person shooter set in a prehistoric world destroyed by cataclysmic events. Players face relentless melee monsters while navigating atmospheric ruins. The game combines fast-paced combat, dynamic animations, and strategic gameplay to immerse players in a thrilling survival experience.

Project Info

  • Time Frame: 3 months
  • Engine: Unreal Engine 4.26
  • Genre: Third-Person Shooter
  • Game Mode: Single-player
  • Platform: Windows (PC)

Key Features

  • Combat System Engage in intense battles with a responsive combat system featuring ranged weapons, melee interactions, and realistic recoil animations.
  • Enemy AI Designed intelligent enemies powered by Behavior Trees, enabling patrol, chase, and attack behaviors that challenge players dynamically.
  • Advanced Animation Systems Implemented features like Blendspaces, Aim Offsets, and Inverse Kinematics for fluid and lifelike character movements.
  • Dynamic Environments Explore destructible, prehistoric-themed levels with immersive lighting, atmospheric effects, and rich environmental storytelling.
  • Interactive HUDs An adaptive HUD displays dynamic crosshairs, health indicators, and weapon stats, reacting seamlessly to player actions.
  • Weapon Mechanics Equipped with automatic and semi-automatic gunfire, reload systems, weapon-specific effects, and bullet trails to enhance combat realism.
  • Explosive Gameplay Elements Integrated explosives with impactful visual effects and area-of-effect damage, adding a strategic edge to battles.
  • Inventory Systems Developed an intuitive inventory system for managing and equipping weapons, complete with rarity indicators and dynamic attributes.
  • Visual & Audio Effects Enhanced immersion with post-process materials, dynamic glow effects, weapon trails, and environmental soundscapes.

Programming Concepts Applied

  • AI Programming Utilized Behavior Trees and Blackboard Components for enemy decision-making and navigation mesh for pathfinding.
  • Animation Techniques Seamlessly blended animations using Animation Blueprints, Montages, and Inverse Kinematics.
  • Gameplay Optimization Applied object-oriented programming principles and optimized material effects for high-performance gameplay.
  • Dynamic Feedback Integrated visual and audio feedback systems, including adaptive crosshairs, hit reactions, and damage indicators.

Challenges & Solutions

  • AI Design Balanced enemy difficulty using debugging tools and fine-tuned behavior trees for realistic engagement.
  • Crosshair Feedback Designed crosshairs that adapt dynamically to player actions, enhancing responsiveness and accuracy.
  • Performance Tuning Optimized particle systems and dynamic materials to ensure smooth performance in complex scenes.

Acknowledgments

Countdown Carnage

About

Countdown Carnage is a high-intensity FPS game set in a dark, dystopian city where players face off against relentless flying drones. With a ticking timer, players must eliminate all drones before time runs out, earning time bonuses with every successful kill.

Project Info

  • Time Frame: 3 weeks
  • Engine: Unreal Engine 5.3
  • Genre: First-Person Shooter (FPS)
  • Game Mode: Single-player
  • Platform: Windows (PC)

Key Features

  • Combat System Responsive and satisfying firearm combat, enhanced with immersive sound effects and visually striking particle systems.
  • Enemy AI Aggressive drones with AI-driven behavior that detect and attack players upon sight or provocation, providing dynamic challenges.
  • User Interface (UI) A functional HUD featuring a countdown timer, health bar, and score display ensures players stay informed in the heat of battle.

Game Mechanics

  • Timer-Based Gameplay The pressure of a countdown timer drives players to act swiftly and strategically.
  • Dynamic Rewards Time bonuses are granted for every drone destroyed, prolonging the player's survival.
  • Pickups Health and ammo pickups scattered throughout the map add an extra layer of strategy.

Visuals and Audio

  • Moody, atmospheric visuals combined with immersive audio effects create a tense and engaging experience.

Programming Concepts Applied

  • Blueprint Scripting Built entirely with Unreal Engine 5 Blueprints, demonstrating advanced visual scripting proficiency.
  • AI Logic Drones are powered by AI logic designed in Blueprints, enabling them to detect, pursue, and attack players dynamically.
  • HUD Design Crafted a fully functional HUD to display essential gameplay elements, including timers and health bars.

Challenges & Solutions

  • AI Pathfinding Fine-tuned navigation meshes to ensure drones efficiently navigate the cityscape.
  • Timer System Implemented a responsive countdown timer that dynamically adjusts with gameplay events.
  • Optimization Debugged and streamlined Blueprint scripts for smooth performance and gameplay.

Acknowledgments

  • Special thanks to the GameDev.tv Team and Stephen Ulibarri for their guidance through the course "Unreal Engine 5 Blueprints: First Person Shooter (FPS)".

Battle Blasters (Ongoing)

About

Battle Blasters is a fast-paced multiplayer shooter in a cartoonish world, featuring dynamic weapon pickups like sniper rifles, assault rifles, and rocket launchers. Players engage in thrilling deathmatch and capture the flag game modes while competing to be the last one standing or securing the flag. The game boasts optimized network code, smooth matchmaking, and lag compensation for an immersive multiplayer experience.

Project Info

  • Time Frame: Ongoing
  • Engine: Unreal Engine 5.2
  • Genre: Multiplayer Shooter
  • Game Mode: Deathmatch, Capture the Flag
  • Platform: Windows (PC)

Key Features

  • Multiplayer Gameplay The game supports seamless online multiplayer, with matchmaking and replication techniques ensuring smooth, fast-paced gameplay for multiple players.
  • Weapon Pickups Players can pick up a variety of weapons scattered across the map, including pistols, SMGs, shotguns, sniper rifles, rocket launchers, and grenades, adding strategic depth to the gameplay.
  • Game Modes Battle Blasters offers multiple game modes, including Deathmatch and Capture the Flag, each with unique challenges and competitive elements.
  • User Interface (UI) The game features a comprehensive HUD with health, shield, and ammo bars, alongside a score display and game timers to track progress.

Game Mechanics

  • Weapon System A variety of weapons are available for players to pick up throughout the map, each with unique characteristics and usage.
  • Weapon Pickups Health, shield, speed buffs, and ammo pickups are scattered across the map to aid players during battle, adding a layer of strategy.
  • Matchmaking & Replication The game utilizes advanced matchmaking for player connection and replication techniques for fast-paced action, ensuring smooth multiplayer sessions.
  • Lag Compensation Client-side prediction and server-side rewind techniques are implemented to minimize the effects of lag and ensure smooth gameplay.

Programming Concepts Applied

  • Networking Used Unreal Engine's online subsystem to implement multiplayer matchmaking and optimize network performance, ensuring fluid gameplay for multiple players.
  • Replication Implemented replication techniques to sync player actions and game events, making sure that all players experience the game in real-time.
  • Game Modes & States Developed custom match states and game modes to manage gameplay flow, including countdown timers, warm-up stages, and winner announcements.

Challenges & Solutions

  • Lag Compensation Overcame lag-related challenges by implementing client-side prediction and server-side rewinding to maintain smooth gameplay even with high latency.
  • Matchmaking Optimization Ensured optimized matchmaking and player connections by programming custom online subsystems to handle player sessions and matchmaking logic.
  • Weapon Balance Fine-tuned weapon behavior and balance to ensure a fun, fair experience for all players, adjusting stats for competitive gameplay.

Acknowledgments

  • Special thanks to Stephen Ulibarri for his Unreal Engine 5 C++ Multiplayer Shooter course, which provided the foundation for many of the networking techniques used in this project.
