Crimson Reign

About

Crimson Reign is a third-person melee combat prototype developed in Unreal Engine 4 using C++. The project focuses on implementing core gameplay systems including combat interactions, AI behavior, player damage handling, and interactive world elements. It served as my first full gameplay project where I built reusable gameplay classes, enemy logic, and player combat mechanics using Unreal's gameplay famework.

Project Info

  • Time Frame: 2 months
  • Engine: Unreal Engine 4.25
  • Genre: Hack-and-Slash
  • Game Mode: Single-player
  • Platform: Windows (PC)

Core Systems Implemented

  • Meele Combat System Implemented player attack handling, hit detection, damage application, and animation-triggered combat events.
  • Enemy AI Behavior Created patrol, chase, and attack behaviors using Behavior Trees, Blackboard, and navigation meshes.
  • Player Health and Damage Built a modular health syste, supporting damage calculation, death handling, and combat feedback.
  • Interactive Environment Programmed triggers, moving platforms, and timed interactions using C++ gameplay actors.
  • HUD and Player Feedback Implemented player UI elements including health bars and combat indicators integrated with gameplay events.

Technical Challenges

  • AI Pathfinding Integrating effective enemy pathfinding with navigation meshes and ensuring smooth navigation during combat and exploration.
  • Combat Feedback Balancing combat feedback (e.g., hit reactions, damage indicators) to ensure a satisfying player experience.
  • Debugging & Optimization Used logging and sphere drawing for debugging gameplay features like AI pathfinding and damage systems to ensure correct behavior during testing.

Apex Ruin

About

Apex Ruin is a third-person shooter prototype developed to explore player combat systems, AI behavior, and animation integration in Unreal ENgine 4.

The project focused on implementing gameplay mechanics in C++ and Blueprints, including combat logic, enemy behavior, animation systems, and player feedback loops.

Project Info

  • Time Frame: 3 months
  • Engine: Unreal Engine 4.26
  • Genre: Third-Person Shooter
  • Game Mode: Single-player
  • Platform: Windows (PC)

Core Systems Implemented

  • Combat System Implemented hitscan weapon firing, recoil handling, and reload logic using Blueprint and C++ interaction.
  • Enemy AI Created AI behaviors using Behavior Trees, including patrol, detection, chase, and attack states with navigation mesh pathfinding.
  • Animation System Integrated animation blueprints, blendspaces, aim offsets, and inverse kinematics to synchronize aiming and movement.
  • HUD & Feedback Implemented dynamic crosshair expansion, health UI, and hit reaction feedback to communicate player actions.
  • Inventory and Weapons Built weapon switching and inventory management logic with configurable weapon properties.

Technical Challenges

  • AI Tuning Enemies initially over-aggressively chased the player. Adjusted perception radius and attack cooldown logic to create balanced combat.
  • Animation Synchronization Faced issues with aim direction mismatching camera orientation. Resolved using Aim Offset blending and controller rotation alignment.
  • Performance Particle and material effects reduced performance during combat. Optimized update frequency and reduced dynamic material updates.

Repository

  • Check out the repository at GitHub for more details and source code.

Battle Blasters (Ongoing)

About

Battle Blasters is a multiplayer shooter prototype focused on implementing server-authoritative networking and gameplay synchronization in Unreal Engine 5.

The project explores player replication, match state management, and latency handling in real-time multiplayer gameplay.

Project Info

  • Time Frame: Ongoing
  • Engine: Unreal Engine 5.2
  • Genre: Multiplayer Shooter
  • Game Mode: Deathmatch, Capture the Flag
  • Platform: Windows (PC)

Networking Systems

  • Replication Implemented replicated player movement, firing, and state variables using Unreal networking framework.
  • Match Flow Designed game state transactions including lobby, warmup, match start, and match end handling.
  • Lag Compensation Implemented client-side prediction and server authority validation to reduce perceived latency.
  • Weapon Synchronization Server validated hits and synchronized damage events across connected clients.
  • Online Sessions Integrated Unreal Online Subsystem for session creation, joining, and matchmaking.

Technical Challenges

  • Network Desync Players experienced hit registeration delay. Resolved using server validation and corrected timing of RPC calls.
  • Latency Implemented prediction and reconciliation to maintain smooth player movement under higher ping.
  • Match Stability Handled player disconnections and ensured match state remained consistent across clients.

  • Check out the repository at GitHub for more details and source code.

About
